# Campaign Command Center — Implementation Plan
_Aloha Health Hub · March 2026_

---

## What This Is

A standalone campaign management tool that connects to your Supabase DB, tracks every outreach touch per listing, sends emails via Resend, and gives you an interactive command center to manage the whole operation. You tell Claude what to do ("send the next batch to Big Island," "show me who replied," "draft follow-ups for Maui non-responders") and Claude executes it.

This is NOT a generic CRM. It's purpose-built for this specific two-phase campaign and wired directly into your existing data model.

## 1. Database Schema

### `campaign_outreach` table (one row per listing being contacted)

```sql
CREATE TABLE campaign_outreach (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  listing_id      uuid NOT NULL,
  listing_type    text NOT NULL CHECK (listing_type IN ('practitioner', 'center')),

  -- Denormalized from listing for fast dashboard reads
  name            text,
  email           text,
  phone           text,
  island          text,
  city            text,
  modalities      text[],
  website_url     text,
  tier            text,        -- current tier: free/premium/featured
  has_owner       boolean,     -- true if listing has been claimed (owner_id IS NOT NULL)

  -- Campaign segmentation
  segment         text NOT NULL CHECK (segment IN (
    'unclaimed',           -- Phase 1: no owner_id
    'claimed_has_website', -- Phase 2 Track A
    'claimed_no_website',  -- Phase 2 Track B
    'bundle_prospect',     -- Phase 2 Track C
    'upgraded',            -- Already converted — stop outreach
    'excluded'             -- Opted out or bad data
  )),
  phase           text CHECK (phase IN ('phase1', 'phase2')),

  -- Pipeline status (the core tracking field)
  status          text NOT NULL DEFAULT 'not_contacted' CHECK (status IN (
    'not_contacted',     -- Haven't reached out yet
    'email_queued',      -- In the next send batch
    'email_1_sent',      -- First email sent
    'email_1_opened',    -- First email opened (Resend webhook)
    'email_2_sent',      -- Follow-up sent
    'replied',           -- They responded (manual update)
    'call_scheduled',    -- Phone follow-up planned
    'called',            -- Phone call made
    'sms_sent',          -- SMS follow-up sent
    'claimed',           -- They claimed their listing (Phase 1 success)
    'upgraded',          -- They upgraded to Premium/Featured (Phase 2 success)
    'bundle_sold',       -- Website bundle purchased (Phase 2 success)
    'not_interested',    -- Declined / unsubscribed
    'bad_contact',       -- Email bounced / phone invalid
    'no_response'        -- Exhausted outreach, no reply
  )),

  -- Outreach history
  email_1_sent_at     timestamptz,
  email_1_opened_at   timestamptz,
  email_1_clicked_at  timestamptz,
  email_1_template    text,
  email_2_sent_at     timestamptz,
  email_2_template    text,
  call_at             timestamptz,
  call_notes          text,
  sms_at              timestamptz,
  reply_at            timestamptz,
  reply_summary       text,

  -- Conversion tracking
  claimed_at          timestamptz,
  upgraded_at         timestamptz,
  upgraded_to         text,  -- 'premium' | 'featured' | 'essentials_bundle' | 'standard_bundle' | 'pro_bundle'
  revenue_monthly     numeric(10,2),  -- MRR from this conversion
  revenue_onetime     numeric(10,2),  -- One-time (website bundle)

  -- Meta
  notes               text,
  priority            integer DEFAULT 0,  -- Higher = contact sooner
  batch_id            text,      -- Which send batch this was part of
  created_at          timestamptz DEFAULT now(),
  updated_at          timestamptz DEFAULT now()
);

-- Indexes for dashboard queries
CREATE INDEX idx_campaign_outreach_segment ON campaign_outreach(segment);
CREATE INDEX idx_campaign_outreach_status ON campaign_outreach(status);
CREATE INDEX idx_campaign_outreach_island ON campaign_outreach(island);
CREATE INDEX idx_campaign_outreach_listing ON campaign_outreach(listing_id);
```

### `campaign_emails` table (log of every email sent)

```sql
CREATE TABLE campaign_emails (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  outreach_id     uuid REFERENCES campaign_outreach(id),
  resend_id       text,         -- Resend message ID for tracking
  to_email        text NOT NULL,
  to_name         text,
  subject         text NOT NULL,
  template        text NOT NULL, -- 'phase1_claim' | 'phase2_track_a' | 'phase2_track_b' | 'phase2_track_c' | 'follow_up'
  body_preview    text,          -- First 200 chars of body
  status          text DEFAULT 'sent' CHECK (status IN ('sent', 'delivered', 'opened', 'clicked', 'bounced', 'complained')),
  sent_at         timestamptz DEFAULT now(),
  delivered_at    timestamptz,
  opened_at       timestamptz,
  clicked_at      timestamptz,
  bounced_at      timestamptz,
  bounce_reason   text
);

CREATE INDEX idx_campaign_emails_outreach ON campaign_emails(outreach_id);
CREATE INDEX idx_campaign_emails_resend ON campaign_emails(resend_id);
```

## 2. Python Pipeline Scripts

All scripts go in `pipeline/scripts/` alongside the existing pipeline.

### `40_campaign_init.py` — Seed the outreach table from existing listings

**What it does:**
- Queries all practitioners and centers from the DB
- For each listing, determines the segment:
  - `owner_id IS NULL` → `unclaimed` (Phase 1 target)
  - `owner_id IS NOT NULL AND tier = 'free' AND website_url IS NOT NULL` → `claimed_has_website`
  - `owner_id IS NOT NULL AND tier = 'free' AND (website_url IS NULL OR website_url = '')` → `claimed_no_website`
  - `tier IN ('premium', 'featured')` → `upgraded` (skip)
- Inserts into `campaign_outreach` with denormalized fields
- Sets `priority` based on: has email (high) > has phone (medium) > neither (low)
- Idempotent: skips listings already in the table (by listing_id)

**Usage:**
```bash
cd pipeline
python scripts/40_campaign_init.py
python scripts/40_campaign_init.py --island big_island  # Single island
python scripts/40_campaign_init.py --refresh             # Re-sync segments from current DB state
```

### `41_campaign_segment.py` — Re-segment and generate send batches

**What it does:**
- Re-reads practitioner/center data to update segments (a listing might have been claimed since init)
- Generates a "send batch" — selects the next N contacts to email, filtered by:
  - Island (required)
  - Segment (required)
  - Status = 'not_contacted' (for new sends) or status in ('email_1_sent', 'no_response') for follow-ups
  - Priority (highest first)
  - Rate limiting: max 25-30 per batch (Resend deliverability best practice)
- Sets status to 'email_queued' for the batch
- Outputs batch to `output/batch_{timestamp}.json` for review before sending

**Usage:**
```bash
python scripts/41_campaign_segment.py --island big_island --segment unclaimed --limit 25
python scripts/41_campaign_segment.py --island maui --segment claimed_no_website --limit 20
python scripts/41_campaign_segment.py --island oahu --segment claimed_has_website --follow-up  # Follow-ups only
```

### `42_campaign_send.py` — Send emails via Resend

**What it does:**
- Reads a batch file (from 41) or queries 'email_queued' contacts directly
- For each contact:
  - Selects the right email template based on segment
  - Personalizes: [Name], [city], [island], [modality], profile URL
  - Sends via Resend API
  - Logs to `campaign_emails` table
  - Updates `campaign_outreach` status to 'email_1_sent' (or 'email_2_sent')
  - Records `email_1_sent_at` timestamp
- Rate limits: 2-second delay between sends (Resend best practice for cold email)
- Handles bounces: marks as 'bad_contact' immediately
- Dry-run mode: generates emails without sending

**Usage:**
```bash
python scripts/42_campaign_send.py --batch output/batch_20260318.json
python scripts/42_campaign_send.py --batch output/batch_20260318.json --dry-run  # Preview only
python scripts/42_campaign_send.py --queued  # Send all 'email_queued' contacts
```

**Environment variables:**
```bash
RESEND_API_KEY=re_xxxx           # From Resend dashboard
RESEND_FROM=aloha@hawaiiwellness.net
RESEND_REPLY_TO=aloha@hawaiiwellness.net
```

### `43_campaign_sync.py` — Sync email statuses from Resend + DB changes

**What it does:**
- Pulls email delivery/open/click events from Resend API
- Updates `campaign_emails` table with delivery timestamps
- Updates `campaign_outreach` with latest status (opened, clicked)
- Checks for newly claimed listings: if `practitioners.owner_id` changed from NULL → UUID, update outreach status to 'claimed' and re-segment
- Checks for tier upgrades: if `practitioners.tier` changed to 'premium' or 'featured', mark as 'upgraded'
- Outputs a summary: "12 delivered, 4 opened, 1 clicked, 2 bounced, 3 claimed since last sync"

**Usage:**
```bash
python scripts/43_campaign_sync.py               # Full sync
python scripts/43_campaign_sync.py --since 24h    # Last 24 hours only
```

### `44_campaign_export.py` — Export data for the dashboard

**What it does:**
- Queries `campaign_outreach` + `campaign_emails` + current practitioners/centers data
- Generates `campaign_data.json` with:
  - Summary stats (total contacts, by segment, by status, by island)
  - Pipeline funnel numbers (not_contacted → email_sent → opened → replied → claimed → upgraded)
  - Full contact list with all fields (for the table view)
  - Email log (for the detail view)
  - Revenue tracking (MRR from upgrades, one-time from bundles)
  - Timeline of activity (sends per day, claims per day)
- Dashboard reads this file — no direct DB connection needed in the HTML

**Usage:**
```bash
python scripts/44_campaign_export.py
# Output: output/campaign_data.json (read by dashboard)
```

### `run_campaign.sh` — Orchestrator

```bash
#!/bin/bash
# Daily campaign workflow
cd pipeline
echo "=== Syncing statuses ==="
python scripts/43_campaign_sync.py
echo "=== Refreshing segments ==="
python scripts/40_campaign_init.py --refresh
echo "=== Exporting dashboard data ==="
python scripts/44_campaign_export.py
echo "=== Done — open campaign-dashboard.html ==="
```

## 4. Resend Setup

### Steps to get running

1. **Create Resend account** at resend.com (free tier: 3,000 emails/month, 100/day)

2. **Verify domain** — add DNS records for hawaiiwellness.net:
   - SPF: `v=spf1 include:resend.com ~all`
   - DKIM: Resend provides the key
   - DMARC: `v=DMARC1; p=none;`
   (These go in your domain's DNS settings — Cloudflare, Namecheap, wherever hawaiiwellness.net is managed)

3. **Get API key** — Resend Dashboard → API Keys → Create

4. **Set environment variable:**
   ```bash
   # In pipeline/.env
   RESEND_API_KEY=re_xxxxxxxxxxxx
   ```

5. **Test send:**
   ```bash
   python scripts/42_campaign_send.py --test --to marcuswoo@gmail.com
   ```

### Webhook for open/click tracking

Resend can POST delivery events to an endpoint. Two options:
- **Option A (recommended for now):** Poll via API in `43_campaign_sync.py` — no webhook setup needed, just run the sync script
- **Option B (later):** Set up a Supabase Edge Function to receive Resend webhooks for real-time status updates

Start with Option A. It's simpler and you control when syncs happen.

## 6. How You Work With Claude

The command center is a read-only dashboard. All ACTIONS go through Claude.

### Daily Workflow

**Morning:**
```
You: "Run the daily sync and show me what happened overnight"
Claude: [runs 43_campaign_sync.py + 44_campaign_export.py]
Claude: "4 emails opened, 1 reply from Dr. Keoni in Kona (interested in Premium),
         2 listings claimed overnight (Paia yoga teacher, Hilo chiropractor),
         1 bounce (bad email for Waimea acupuncturist)"
```

**Sending a batch:**
```
You: "Queue the next 25 Big Island unclaimed listings for Phase 1 email"
Claude: [runs 41_campaign_segment.py --island big_island --segment unclaimed --limit 25]
Claude: "25 contacts queued. Here are the first 5:
         1. Dr. Keoni Makua — Acupuncture, Kailua-Kona
         2. Sarah Leilani — Massage, Hilo
         ...
         Want me to send them?"
You: "Yes, send"
Claude: [runs 42_campaign_send.py --queued]
Claude: "25 emails sent. 0 bounces. Dashboard refreshed."
```

**Handling replies:**
```
You: "Sarah Leilani replied — she's interested but wants to know about websites"
Claude: [updates status to 'replied', moves segment to 'bundle_prospect',
         adds note, runs export]
Claude: "Updated. She's now in the website bundle pipeline. Want me to draft
         a personalized response or send the Track C template?"
```

**Checking progress:**
```
You: "How's the Big Island campaign looking?"
Claude: [reads campaign_data.json]
Claude: "Big Island Phase 1: 120 contacted, 34% open rate, 12 claimed (10%).
         Phase 2: 8 contacted, 2 upgraded to Premium so far.
         Next batch ready: 45 unclaimed listings remaining."
```

**Segment moves:**
```
You: "These 3 practitioners claimed their listings but I can see they have
      websites now. Move them to Track A."
Claude: [updates segment, runs export]
```
